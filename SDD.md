# Software Design Document (SDD) - CU Orbit

## 1. System Architecture
### 1.1 Overview
The system follows a Client-Server architecture. The Android client communicates with an Express.js REST API via HTTP/HTTPS.

### 1.2 Tech Stack
- **Frontend**: Kotlin, Android Jetpack (Navigation, ViewModel, LiveData), Coroutines, Material Design.
- **Backend**: Node.js, Express, Sequelize ORM.
- **Database**: MySQL.
- **Media**: Multer for local server storage.

## 2. Component Design
### 2.1 Android App Structure
- **UI Components**: Fragments for Home, DMs, Status, Calls, and Profile.
- **Data Layer**: Retrofit interfaces for networking, Repositories for data abstraction.
- **State Management**: ViewModels to handle UI-related data and lifecycle-aware updates.

### 2.2 Server Structure
- **Middleware**: Request logging, static file serving (`/uploads`), and DB health checks.
- **Routes**: Specialized controllers for Auth, Channels, Messages, Status, and Calls.
- **Models**: Sequelize definitions for Users, Channels, Messages, Reactions, etc.

## 3. Database Design
### 3.1 Schema Summary
- **Users**: Unique phone numbers, names, and profile bios.
- **Channels**: Group metadata and member counts.
- **Messages**: Core data including type (text/image/voice), media URLs, and read status.
- **Statuses**: Short-lived media entries with an `expiresAt` timestamp.
- **Calls**: Call history logs for tracking engagement.

## 4. Key Logic Flows
### 4.1 Shared DM Room Generation
To ensure both users see the same history, the DM Room ID is generated by sorting their phone numbers numerically: `Math.min(p1, p2) + "_" + Math.max(p1, p2)`.

### 4.2 Media Handling
1. Client captures photo/audio.
2. File is uploaded to server `/api/upload`.
3. Server returns a permanent URL.
4. Client sends a message referencing this URL.
5. Recipient's client downloads/renders media using Coil or MediaPlayer.

## 5. Security & Deployment
- **CORS**: Configured for cross-origin resource sharing.
- **Nginx Proxy**: Used in production to map HTTPS traffic to the Node.js port (3000).
- **Gitignore**: Protecting sensitive environment files and binary artifacts.
